一、核心目标
--------------------------------------------------------------------------------
在不修改大模型权重的前提下，通过外挂记忆系统提升大模型的：
  1. 长期记忆能力（解决上下文窗口限制）
  2. 联想推理能力（解决孤立RAG无法推导的问题）
  3. 元认知能力（知道自己知道什么、不知道什么）
  4. 自我进化能力（推理结论反哺知识库）

与现有技术的本质区别：
  - 普通RAG：关键词/向量相似度检索，无推理，无结构
  - MemGPT：层级记忆管理，但无联想推导
  - 知识图谱：有结构，但检索靠精确匹配，无语义泛化
  - 本系统：向量检索 + 图谱推导 + 分层结构 + 验证机制 + 自我进化


二、记忆节点结构
--------------------------------------------------------------------------------
每个记忆节点包含：

  node_id        : 唯一标识
  content        : 内容文本
  memory_type    : 类型（事实/规律/经验/推断/元认知）
  abstraction_level : 抽象层级（0=具体实例 ~ 1=普遍规律）
  context_tags   : 语境标签列表（如 ["生存", "动物行为"]）
  relations      : 关联关系列表
  confidence     : 置信度（0~1）
  access_count   : 访问次数（热度）
  created_at     : 创建时间
  last_accessed  : 最后访问时间
  source         : 来源（human_input/inference/observation）


三、记忆分层结构（由内到外）
--------------------------------------------------------------------------------
  Layer 0 - 本质特征层：事物最核心的属性（如"哺乳动物=恒温+哺乳+有毛"）
  Layer 1 - 物理规律层：不可违背的规律约束（如"生物需要能量维持生命"）
  Layer 2 - 外延关系层：与其他概念的关联（如"猫 is-a 哺乳动物"）
  Layer 3 - 标签辅助层：场景/情感/领域标签（用于语境过滤）
  Layer 4 - 验证权重层：每条关系的可靠度权重（支持渐进调整）


四、元关系类型（约25种固定类型）
--------------------------------------------------------------------------------
  结构关系：is-a（上位）、part-of（组成）、instance-of（实例）
  因果关系：causes（导致）、enables（使能）、prevents（阻止）
  功能关系：used-for（用途）、achieved-by（实现方式）
  相似关系：similar-to（相似）、analogous-to（类比）、opposite-of（对立）
  时序关系：precedes（先于）、follows（后于）、co-occurs-with（共现）
  空间关系：located-in（位于）、contains（包含）
  属性关系：has-property（具有属性）、measured-by（度量方式）
  验证关系：supports（支持）、contradicts（矛盾）、requires（依赖）
  演化关系：evolves-from（演化自）、derived-from（派生自）
  元认知：uncertain-about（不确定）、inferred-from（推断来源）

设计原则：
  - 固定元关系保证图谱可推理（关系类型可传播）
  - 动态外延标签保证灵活性（不同语境下权重不同）
  - 二者结合而非二选一


五、双路径检索机制
--------------------------------------------------------------------------------
  路径B（快捷边，速度优先）：
    向量相似度 -> 直接命中高相关节点
    优点：速度快
    缺点：无法推导，遗漏隐含关联

  路径A（元关系推导，泛化优先）：
    1. 向量检索锚点节点
    2. 沿元关系边扩展（广度优先，可配置层数）
    3. 缺口检测：发现推理链断裂处
    4. 反向推导：从目标倒推缺失节点
    5. 四重验证筛选结果

两路径结合：B路径快速命中 + A路径补全推导链


六、双重筛选机制
--------------------------------------------------------------------------------
  第一重：覆盖度（静态）
    - 候选节点覆盖查询关键概念的比例
    - 防止检索结果偏离主题

  第二重：语境相关度（动态）
    - 当前对话语境 vs 节点的语境标签
    - 同一节点在不同语境下相关度不同
    - 例："心跳加速"在"运动"语境相关，在"恐惧"语境也相关，但权重不同


七、四重约束验证
--------------------------------------------------------------------------------
  验证1 - 物理规律相容：结论不违背Layer1的物理/生物规律
  验证2 - 目标特征符合：结论与查询目标的本质特征匹配
  验证3 - 内部无矛盾：推理链内部各节点间无逻辑冲突
  验证4 - 背景无冲突：结论与已知背景知识无冲突

全部通过 -> 输出结论，置信度高
部分通过 -> 输出结论，标注不确定项
全部失败 -> 报告"无法推导"，标注知识缺口


八、渐进式权重调整
--------------------------------------------------------------------------------
  目标：防止单向错误积累，保留自我纠正能力

  机制：
    - 每次验证通过 -> 相关关系权重小幅上升（+delta）
    - 每次验证失败 -> 相关关系权重小幅下降（-delta）
    - 权重变化有上下限（不会到达0或1的极端）
    - 多次矛盾触发"冲突标记"，等待人工审核或更多证据

  注意：权重调整量要保守（delta较小），避免噪声数据影响核心知识


九、推理结论自动存储（用户提出的关键改进）
--------------------------------------------------------------------------------
  机制：
    验证通过的推理结论 -> 自动作为新节点存入知识图谱
    存储时记录：来源节点、推理路径、置信度、时间戳

  对三大瓶颈的帮助：

  瓶颈1 - 知识图谱自动构建质量差：
    推理结论补充了人工标注遗漏的隐含知识
    每次对话都在悄悄丰富知识库
    -> 部分解决，但初始质量仍依赖人工

  瓶颈2 - 抽象层级标注困难：
    推理结论继承来源节点的抽象层级
    多次从具体推导出的共性结论 -> 自动提升抽象层级
    -> 有帮助，形成自底向上的抽象层级发现机制

  瓶颈3 - 验证可靠性：
    推理结论需通过四重验证才能存储（不存储未验证结论）
    存储时标注置信度，支持后续被推翻
    -> 有帮助，形成验证-存储的闭环


十、三个待实现的改进方向（下次继续的重点）
--------------------------------------------------------------------------------

  改进1：向量方向判断关系类型
  -----------------------------------------------
  核心思想：
    两个节点的向量之差（方向）编码了它们的关系信息
    类似word2vec的"king - man + woman = queen"

  实现思路：
    1. 收集已知关系对（nodeA, relation, nodeB）的向量
    2. 计算向量差 delta = vec(B) - vec(A)
    3. 对每种关系类型，学习一个"典型方向向量"
    4. 新节点对的向量差 -> 与各关系类型典型方向计算余弦相似度
    5. 相似度最高的关系类型 -> 自动标注候选关系
    6. 置信度低的候选关系标记为"待验证"

  价值：
    - 大幅减少人工标注工作量
    - 发现人工遗漏的隐含关系
    - 支持知识图谱自动扩展

  难点：
    - 需要足够多的已知关系对作为训练数据
    - 同一向量差可能对应多种关系（歧义问题）
    - 不同语义空间中方向的稳定性待验证


  改进2：语境-层级映射表
  -----------------------------------------------
  核心思想：
    同一个概念在不同语境下应该激活不同的抽象层级
    例："猫" 在"宠物"语境激活实例层，在"生物进化"语境激活规律层

  实现思路：
    1. 建立语境标签 -> 推荐抽象层级范围的映射表
       例：{"日常对话": [0, 0.3], "科学分析": [0.6, 1.0], "工程应用": [0.3, 0.7]}
    2. 检索时先识别当前语境标签
    3. 根据映射表过滤候选节点的抽象层级范围
    4. 同一查询在不同语境返回不同粒度的结果

  价值：
    - 避免在日常对话中返回过于抽象的规律（显得生硬）
    - 避免在科学分析中返回过于具体的实例（缺乏泛化）
    - 实现"说话看对象"的语境适配能力

  难点：
    - 语境识别本身就是难题（需要NLU能力）
    - 映射表的粒度和覆盖度需要大量调试
    - 语境边界模糊时的处理策略


  改进3：新信息冲突分级处理
  -----------------------------------------------
  核心思想：
    新信息与已有知识冲突时，不能简单覆盖或拒绝
    需要根据冲突性质分级处理

  四级冲突处理方案：
    Level 1 - 补充（无冲突）：
      新信息是已有知识的扩展 -> 直接添加新节点/关系
      例：已知"猫会爬树"，新信息"猫会游泳" -> 补充属性

    Level 2 - 细化（局部冲突）：
      新信息与某个具体实例冲突，但不违背规律层
      -> 添加条件限定，保留两者
      例："鸟会飞"（规律）vs"企鹅不会飞"（实例）
      -> 规律层保留，添加"except"关系到企鹅节点

    Level 3 - 修正（规律级冲突）：
      新信息与规律层冲突，且有高置信度支持
      -> 降低原规律置信度，标记为"待审核"
      -> 同时存储新信息，标注来源和证据
      例：新的科学发现推翻旧理论

    Level 4 - 悖论（根本矛盾）：
      新信息与本质特征层冲突，且无法调和
      -> 标记为悖论，不修改现有知识
      -> 记录冲突细节，等待人工裁决
      例：直接矛盾的两条公理

  价值：
    - 避免知识库被错误信息轻易覆盖
    - 避免知识库僵化拒绝所有更新
    - 实现像人类一样的"有保留地接受新信息"

十五、三大遗留问题分析（2026-03-14）
--------------------------------------------------------------------------------

  问题1：词汇 Overlap 精度有限
  ─────────────────────────────
  现象：
    问题响应度（维度①）依赖词汇字面命中，
    问题与节点语义相关但用词不同时得分低（如"多巴胺"vs"奖励信号"）。

  改进方向（可行性分析）：
    建议方案：response_score = max(词汇overlap, step1_top_向量相似度)
    - 可行性：高（已有 vector_search 返回相似度，直接读取即可）
    - 效果上限：受限于当前 embedding 模型精度（nomic-embed-text 对中文领域词的区分度有限）
    - 根本解决：换用领域精调的中文 embedding 模型（如 BGE-M3/text2vec-large-chinese）
    - 结论：词汇兜底方案可大幅改善，但根本解决依赖模型升级，可行

  问题2：输入端元数据问题（"外挂式预训练知识库"）
  ─────────────────────────────────────────────────
  背景：
    新信息进入系统时，essence_features / abstract_level / domain 等字段
    需要人工填写，大模型输入时无法自动补全。

  "外挂式语义知识预置层"方案评估：
    A. 预置基础模式数据（可行，★★★★★）
       预先构建覆盖常见领域的元数据模板库，输入时按语义最近匹配模板自动填充。
       门槛低，立即可做。

    B. 盲区识别+联网填充（中高可行，★★★☆☆）
       检测 essence_features 为空的节点，触发外部知识查询（Wikipedia/知识图谱API）
       自动补全元数据。技术可行，依赖外部服务稳定性。

    C. 后台持续自动标注（中等可行，★★★☆☆）
       用小模型对新入库节点做自动 NER+分类，标注 abstract_level 和 domain。
       瓶颈在于 abstract_level 的标注——它是自定义语义，无现成训练数据。
       需先人工标注50~100条样本，再微调分类器。

    综合结论：
      外挂式知识预置层是可行的独立模块，建议分步实现：
      第一步：A（预置模板库），第二步：C（自动标注小模型），第三步：B（联网补全）。

  问题3：悖论池（paradox_pool）说明
  ────────────────────────────────────
  定义：
    ConflictResolver 中的 List[ConflictReport]，用于暂存无法自动解决的根本性矛盾。

  触发条件：
    同时满足以下条件：
    - 冲突节点的 abstract_level ≥ 8（规律层/本质层，已是知识库的"公理"级别）
    - 新信息的 evidence_strength < 0.8（证据不足以推翻现有公理）
    此时系统判断：无法自动裁决，人工介入。

  当前状态：有存无读
    - 写入路径：_act_paradox() → self.paradox_pool.append(report) ✓
    - 读取路径：无处理/无读取/无清理接口 ✗
    - 影响：悖论只会积累，不会被处理，无法关闭循环

  需要增加的三路出口（下次实现）：
    ① 新信息胜（旧公理被推翻）：
       人工/大模型确认后，降低旧节点权重，写入新节点，从悖论池移除
    ② 旧信息胜（新信息错误）：
       拒绝新信息，在悖论池中标记为"已驳回"，记录驳回原因
    ③ 条件不同，双方共存：
       判定两者适用语境不同（如"鸟会飞" vs "企鹅不会飞"），
       降级为 L1（局部冲突/例外），从悖论池移除，按 L1 处理

  举例：
    场景：知识库中有"本能行为不依赖后天学习"（abstract_level=9，属公理级）
    新输入："章鱼学习开瓶盖的行为表明本能中包含学习成分"（evidence_strength=0.6）
    触发悖论（abstract_level=9 且 evidence_strength<0.8）
    → 加入悖论池，等待人工判定是否走③共存出口（此章鱼案例应走③）

  1. 归谬沙箱裁决逻辑修复 —— associative_engine.py
  ────────────────────────────────────────────────────
  原问题：置信度高≠假设被支持，旧逻辑混淆了"知识库有相关知识"与"假设被支持"。

  修复方案：三路独立信号取代置信度判断
    信号A（结构矛盾）：激活节点与公理之间有 OPPOSITE_TO 边 → 权重×3
    信号B（极性矛盾）：激活节点 INHIBITS 某公理节点 → 权重×2
    信号C（语义覆盖）：假设与公理语义相似+公理被激活
                      → 若公理有FALSE邻居节点则反驳(×1)，否则支持(×2)
    信号D（公理激活）：激活的已确认公理节点（无矛盾边）→ 弱支持(×0.5/个)

  裁决规则：
    反驳得分/(反驳+支持) ≥ 0.60 → refuted
    支持得分/(反驳+支持) ≥ 0.60 → supported
    其余 → inconclusive
  返回字典新增：refute_score / support_score / support_nodes

  2. source_trust 断路修复 —— conflict_resolver.py
